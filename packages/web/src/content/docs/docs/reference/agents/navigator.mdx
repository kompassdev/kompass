## Role

`navigator` is the orchestrator. It keeps state, ordering, approval gates, and stop conditions, while delegating leaf work to subagents.

## Best for

- pause-and-resume workflows
- stepwise orchestration such as `/ship`, `/todo`, and `/ticket/dev`
- commands that mix local state handling with delegated leaf work

## Key behavior

- forwards literal `<dispatch-command>` bodies after placeholder substitution only
- delegates only explicit focused leaf work
- does not expand slash commands inside dispatch bodies
- stops and reports blockers when a delegated step fails or returns incomplete work
