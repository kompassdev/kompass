## Role

`worker` is the default generic worker agent. In the bundled config it has no specialized prompt and only minimal built-in policy beyond normal execution rules.

## Best for

- focused implementation or follow-up work
- direct command execution that does not need planner, navigator, or reviewer specialization
- asking a targeted question when a command allows it and execution is blocked

## Notes

- bundled description: `Generic worker agent.`
- bundled permission: `question: allow`
- no dedicated bundled prompt file is loaded for this role
