## Purpose

Use `/branch` to create a short categorized branch name from current uncommitted work.

## Usage

```text
/branch [context]
```

## Typical flow

- load uncommitted changes with `changes_load`
- stop with `Nothing to branch from` when there is no work
- skip when the current branch already looks like a work branch
- choose a category such as `feature`, `fix`, `refactor`, `docs`, `test`, or `chore`
- create and checkout `<category>/<slug>`, retrying once with a numeric suffix if needed

## Common tools

- `changes_load`
- `bash(git checkout -b)`

## Output

- `Nothing to branch from`
- `Already on work branch: <current-branch>`
- `Created branch: <new-branch>` with the prior branch name
