## Purpose

Use `/dev` for focused implementation work that stops at a PR handoff.

## Usage

```text
/dev <description>
```

## Typical flow

- load ticket context when the argument is a ticket reference
- summarize the request, constraints, and acceptance criteria
- implement the change with focused edits
- run relevant validation
- prepare a PR handoff instead of creating the PR directly

## Common tools

- `ticket_load`
- `read`
- `apply_patch`
- `bash(...)`
