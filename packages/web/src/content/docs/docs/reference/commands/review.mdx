## Purpose

Use `/review` for a local review pass that does not publish comments.

## Usage

```text
/review [base-or-uncommitted]
```

## Typical flow

- load branch or working-tree changes
- inspect the changed files with repository-aware context
- assign a grade and top-line verdict
- return severity-counted findings without mutating the branch or posting comments

## Common tools

- `changes_load`
- `read`
