## Purpose

Use `/rmslop` to clean branch work that looks noisy, inconsistent, or overly AI-shaped.

## Usage

```text
/rmslop [base-or-guidance]
```

## Typical flow

- load branch changes against an optional base branch
- identify slop such as awkward comments, abnormal defensive code, `any` escapes, emoji, or style drift
- make focused cleanup edits
- run relevant validation
- stage and commit the cleanup

- `changes_load`
- `apply_patch`
- `bash(...)`
