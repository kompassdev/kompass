## Purpose

Use `/ship` when you want a fast path from current work to a PR through local orchestration.

## Usage

```text
/ship [base-or-context]
```

## Typical flow

- ensure there is a suitable work branch
- delegate `/commit`
- delegate `/pr/create`
- stop on blockers and report `Nothing to ship` when PR creation finds no shippable work

## Common tools

- `dispatch(/branch)`
- `dispatch(/commit)`
- `dispatch(/pr/create)`
