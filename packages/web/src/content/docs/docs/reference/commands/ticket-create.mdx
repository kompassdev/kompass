## Purpose

Use `/ticket/create` to turn a request or work summary into a tracked issue.

## Usage

```text
/ticket/create <description>
```

## Typical flow

- summarize the request or completed work
- generate a scoped issue title and body
- add checklist sections for the work to track

## Common tools

- `ticket_sync`
