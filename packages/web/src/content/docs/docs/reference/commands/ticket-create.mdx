## Purpose

Use `/ticket/create` to turn the current branch or working-tree changes into a tracked issue.

## Usage

```text
/ticket/create [base-or-context]
```

## Typical flow

- load the current change comparison
- stop if there is no work to summarize
- generate a scoped issue title and body
- add checklist sections for the work to track
- create the issue and assign it to the author

## Common tools

- `changes_load`
- `ticket_sync`
