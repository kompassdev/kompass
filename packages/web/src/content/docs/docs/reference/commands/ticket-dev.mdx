## Purpose

Use `/ticket/dev` when implementation starts from a tracked ticket and should run through shipping.

## Usage

```text
/ticket/dev <ticket-reference>
```

## Typical flow

- load the ticket and its surrounding context
- delegate `/dev` for implementation and validation
- delegate `/branch` for branch naming from the ticket summary
- delegate `/commit-and-push`
- delegate `/pr/create` and return the resulting PR URL

## Common tools

- `ticket_load`
- `dispatch(/dev)`
- `dispatch(/branch)`
- `dispatch(/commit-and-push)`
- `dispatch(/pr/create)`
