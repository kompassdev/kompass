## Purpose

Use `/todo` for structured, pause-and-resume work from a todo file.

## Usage

```text
/todo [todo-file]
```

## Typical flow

- load the todo file
- keep orchestration local
- delegate one focused task at a time
- report what is done, blocked, or next

## Common tools

- `read`
- `task(navigator)`
- `task(worker)`
