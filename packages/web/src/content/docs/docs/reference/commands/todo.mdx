## Purpose

Use `/todo` for structured pause-and-resume work from a checklist file.

## Usage

```text
/todo [@todo-file]
```

## Typical flow

- load `@TODO.md` by default or a provided `@file`
- take the first unchecked item only
- delegate `/ticket/plan`, show the plan, and require explicit approval before implementation
- delegate `/dev`, then `/commit`, then mark the checklist item complete
- repeat until there are no pending tasks left

## Common tools

- `read`
- `question`
- `dispatch-command(/ticket/plan)`
- `dispatch-command(/dev)`
- `dispatch-command(/commit)`
