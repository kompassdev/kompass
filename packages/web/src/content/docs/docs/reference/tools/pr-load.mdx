---
title: pr_load
description: Load normalized PR metadata, review history, and thread context from GitHub.
---

## Purpose

Use `pr_load` when a workflow needs one grounded PR context payload instead of separate GitHub calls.

## Inputs

- `pr` optional; accepts a PR number or URL

## Returns

- `repo`
- `viewerLogin`
- `pr` metadata including title, body, URL, refs, commit count, and author
- `reviews`
- `issueComments`
- `threads`

## Useful for

- `/pr/review`
- `/pr/fix`
