title: ticket_load
description: Load a ticket from GitHub, a file path, or raw text.
---

## Purpose

Normalize ticket input so planning and implementation flows can start from the same shape.

## Inputs

- `source` required
- `comments` optional

## Useful for

- `/ticket/ask`
- `/ticket/dev`
- `/ticket/plan`
- `/ticket/plan-and-sync`
- `/pr/review` when the PR references a ticket
