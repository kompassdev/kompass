description: Create or update a GitHub issue with structured checklist support.
---

## Purpose

Use one tool surface for issue creation, issue updates, and ticket comments.

## Notable inputs

- `title`
- `body`
- `description`
- `labels`
- `assignees`
- `checklists`
- `refUrl`
- `comments`

## Useful for

- `/ticket/create`
- `/ticket/ask`
- `/ticket/plan-and-sync`
- `/pr/create` when ticket auto-creation is needed
